Mockingbird API Documentation

Here is a detailed explanation of the different options for using the Mock decorator:

IdentifierFunctionWill GenerateNotes
Callback@Mock(callback: (faker: Faker) => any)Value from the callback invocationUses the Faker library under the hood, Faker docs are here
Inferred Value@Mock()Random value inferred from the property type
Class@Mock(value: Class)Matching class typePrimitive constructors can be used as well
Absolute Value@Mock(value: string \| boolean \| number \| ObjectLiteral)The exact given value
Enum@Mock(value: { enum: object })Random value from the given enumThe random value is not the key of the enum but the value
Array of Classes@Mock(options: { type: Class, count: number })Array with count items from the given Class

The Class interface looks like this:

export type Class<T = any> = new (...args: any[]) => T;

and represents a 'type' of actual class (not an instance)

Callback#

The first option, probably the most common one, is to pass a callback function that uses the faker argument as the actual faker instance.

So the result of the following code:

import { MockFactory } from 'mockingbird-ts';
class Person {  @Mock(faker => faker.internet.email())  email: string;}
const person = MockFactory(Person).one();console.log(person);

will be

{  email: 'random-email@address.com'}

Inferred Value#

When using the Mock decorator without any value will generate a random value inffered from the property type.

So the result of the following code:

import { MockFactory } from 'mockingbird-ts';
class Person {  @Mock()  serial: string;
  @Mock()  points: number;
  @Mock()  isLucky: boolean;}
const person = MockFactory(Person).one();console.log(person);

will be:

{  serial: 'uirjkcmovf',  points: 64,  isLucky: true}

Type string will generate a 10 characters random string \ Type number will generate a number between 1 to 100 \ Type boolean will of course generate true or false

Class#

Passing a class will generate an object with the matching keys (decorated by the Mock decorator)

So the result of the following code:

import { MockFactory } from 'mockingbird-ts';
class Dog {  @Mock(faker => faker.name.firstName())  name: string;}
class Person {  @Mock()  serial: string;
  @Mock()  points: number;
  @Mock(Dog)  dog: Dog;
  @Mock()  isLucky: boolean;}
const person = MockFactory(Person).one();console.log(person);

Will be:

{  serial: 'uirjkcmovf',  points: 64,  dog: {    name: 'Bucky'  },  isLucky: true}

Absolute Value#

The "Absolute Value" option is pretty strait forward, the generated value from the Mock decorator will the exact same value that has been passed

So the result of the following code:

import { MockFactory } from 'mockingbird-ts';
class Person {  @Mock('John')  serial: string;
  @Mock(78)  points: number;
  @Mock(true)  isLucky: boolean;}
const person = MockFactory(Person).one();console.log(person);

Will be:

{  serial: 'John',  points: 78,  isLucky: true}

Enum#

Passing an enum object to the Mock decorator will generate a random value from the given enum (not a key):

So the result of the following code:

import {MockFactory} from 'mockingbird-ts';
enum Mood {  Happy = 'happy',  Numb = 'numb',  Sad = 'sad'}
class Person {  @Mock({enum: Mood})  mood: string;}
const person = MockFactory(Person).one();console.log(person);

Will be:

{  mood: 'happy'}

Array of Classes#

Just as it is possible to move a class as a parameter, so it is also possible to pass an "array" of classes:

import {MockFactory} from 'mockingbird-ts';
class Dog {  @Mock()  name: string;
  @Mock()  points: number;}
export class DogWalker {  @Mock({ type: Dog, count: 3 })  dogs: Dog[];}
const dogWalker = MockFactory(TestClassWithSingleClass).one();console.log(dogWalker);

Will be:

{  dogs: [    {...},    {...},    {...}  ]}

When each object is basically an instance of a dog class and has the properties 'name' and 'points'.
